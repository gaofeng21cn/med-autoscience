# Publication Route Memory Knowledge

Owner: MedAutoScience
Knowledge role: executable memory rules for reusable publication-route experience
Machine boundary: OPL may consume memory locators and receipt metadata only. MAS owns memory body acceptance, rejection, and writeback authority.

## What Memory Can Contain

Publication-route memory may hold:

- route patterns that previously worked or failed for similar study lines.
- journal-neighbor expectations, reader risks, and reviewer pressure patterns.
- figure/table/story patterns tied to evidence refs.
- common source, provenance, methodology, citation, display, and artifact blockers.
- stop-loss rules, claim downgrade examples, and route-back lessons.
- accepted or rejected writeback records with router receipts.

## How To Use Memory

- Read memory as advisory prior experience, not as current evidence.
- Use memory to generate candidate routes, anticipate reviewer risk, choose robustness checks, and avoid repeated failed paths.
- Link memory use to current refs: study charter, evidence ledger, source readiness, manuscript refs, and controller decisions.
- Preserve memory-derived warnings in receipts so later stages can see why a shortcut was avoided.
- When memory suggests a new claim boundary, route to decision or human gate rather than applying it locally.

## Writeback Rules

A memory writeback proposal must include:

- source stage and receipt refs.
- current study, route, claim, blocker, or reviewer context.
- lesson body and whether it is reusable, study-specific, or rejected.
- evidence refs and failed-path refs that justify the lesson.
- proposed accept/reject outcome, typed blocker if incomplete, and router receipt refs.

Only MAS reviewer/auditor or route authority can accept or reject publication-route memory. Programs and materializers may validate shape/currentness and emit receipts or blockers; they cannot decide memory acceptance.

## Forbidden Uses

- Do not use memory as a substitute for literature search, source readiness, evidence, citation integrity, AI reviewer quality, or publication verdict.
- Do not copy memory body into current study truth without an accept/reject record.
- Do not treat memory locator presence as readiness.
- Do not let OPL generated surfaces read or mutate memory body.

## Route Impact

Memory can justify route-back, stop-loss, robustness checks, manuscript caveats, or reviewer concerns. It cannot by itself advance a stage. Every memory-driven action must still cite current MAS refs and end in an owner receipt, typed blocker, route-back, or human gate.
