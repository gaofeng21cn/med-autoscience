# Source Readiness And Artifact Authority Knowledge

Source readiness asks whether the current data/source/provenance package can support the active study claim. Artifact authority asks whether manuscript, figure, table, supplement, response, package, and delivery outputs are current, rebuildable, and authorized by MAS owner surfaces.

Both are authority boundaries. File presence, generated interface readiness, provider completion, script exit code, or package freshness alone cannot close them. Missing source provenance, stale canonical paper refs, or absent rebuild proof must route back with typed blockers.
