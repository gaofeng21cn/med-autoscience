## MAS-owned skill content companion pattern

Use this companion block when a stage needs a compact template, audit table, or helper checklist next to the skill.

This is MAS-owned companion content, not a runtime, router, scheduler, database, or external tool-selection layer. MAS authority stays in controller decisions, publication evaluation, evidence ledgers, review ledgers, display registries, study contracts, and owner receipts.

Keep it progress-first: create the narrowest useful artifact, bind it to active refs, record missing route-required refs when blocked, and keep moving through the MAS route. Missing companion content is an advisory or platform-repair gap unless the active owner route already requires that exact ref family.

Use local companion files as quest-local working aids:

- `*.template.md`: fillable artifact scaffold for the current stage.
- `*.block.md`: reusable discipline block referenced by a skill.
- Generated evidence, manuscript, display, and review outputs remain in MAS-owned paper/artifact surfaces, not in the skill directory.

Do not import external scripts, runtime state, memory databases, score sheets, checklist verdicts, or tool routers into this layout. If a helper needs executable behavior, implement it through MAS owner code, contracts, and tests first.
