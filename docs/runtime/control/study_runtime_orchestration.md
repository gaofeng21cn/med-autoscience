# Study Runtime Transport Boundary

Owner: `OPL Framework runtime transport + MedAutoScience domain authority`
Purpose: `Define the single AI semantic control boundary for MAS.`
State: `active_runtime_support`
Machine boundary: OPL transports Codex-selected StageRuns; MAS supplies domain artifacts and authority evidence.

当前主链是：

`Codex CLI semantic route -> OPL StageRun transport -> MAS stage prompt/domain helpers -> readable artifact -> Codex CLI next route`

OPL 只持有 queue、attempt、provider、retry/resume transport、identity/currentness 与被动 refs projection。MAS 只持有医学 truth、科学解释、artifact mutation authority、publication/quality verdict、owner receipt、human gate 与真正的 authority/safety blocker。

不存在并行的 MAS transition oracle、outer loop、cycle reducer 或 controller refresh。阴性、无效应、未达预期、部分草稿和失败分析都是可消费 scientific artifacts。Codex 根据证据判断偏差发生在研究问题、假设、evidence、cohort、endpoint、analysis plan、manuscript 或 publication design 哪一层，并可带 lineage route-back 到任意 declared stage。

Retry、review、repair 次数只定义质量预算。预算耗尽后必须保留最佳可读 artifact 并继续；债务只能阻止更高质量或 ready claim，不能阻止 stage progression。
